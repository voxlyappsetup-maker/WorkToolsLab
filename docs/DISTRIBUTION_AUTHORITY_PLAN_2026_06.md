| **More optimize/patch on same data** | Diminishing returns — coverage is largely in place |
| **Young site / limited external signals** | Authority and distribution likely constrain growth |

**Conclusion:** The next lever is **useful external visibility** — referrals, mentions, and trust signals — not another patch loop on stale GSC data.

**What distribution will not do overnight:** Guarantee rankings. It can build awareness, referral traffic, editorial relationships, and long-term authority.

**What distribution should do:** Put high-value pages in front of the right audiences where a link is **earned by usefulness**, not forced.

## 2. Distribution principles

| Principle | Rule |
|-----------|------|
| **Useful-first publishing** | Lead with a tip, checklist, or decision framework; link only when it genuinely helps |
| **No spam** | No mass posting, no copy-paste comments, no drive-by links |
| **No fake engagement** | No fake accounts, upvote rings, or manufactured social proof |
| **No paid links** | No buying backlinks, sponsored link inserts, or PBNs |
| **No irrelevant communities** | Participate only where small-team / freelancer / SaaS tool topics fit |
| **Manual, selective outreach** | Quality over volume; skip if the touchpoint feels forced |
| **Track every action locally** | Use Section 6 template — spreadsheet or markdown log in `docs/` or local notes |

**Hard avoids:** Link farms, low-quality guest-post networks, automated outreach blasts, AI-generated comment spam, misrepresenting “hands-on testing” in outreach.

## 4. Channel strategy

### LinkedIn personal posts

| Item | Guidance |
|------|----------|
| **What to post** | One insight (3–5 sentences) + optional bullet checklist; link in first comment or end if natural |
| **What not to post** | Daily link dumps, engagement bait, “we tested 47 tools” claims |
| **Frequency** | 2–3 posts per week max |
| **Example** | Share “waiting on client” stage tip from freelancers guide → link |
| **Risk** | Low |
| **Tracking** | Section 6 table; note post URL and date |

### LinkedIn comments on relevant posts

| Item | Guidance |
|------|----------|
| **What to post** | Add a substantive comment (2–4 sentences) on PM/freelancer/small-business posts; link only if directly relevant |
| **What not to post** | “Great post! Read my article…” with no value |
| **Frequency** | 3–5 useful comments per week |
| **Example** | On a “small team chaos” post, share weekly task review habit + link to How to Manage Tasks |
| **Risk** | Low–medium (avoid looking promotional) |
| **Tracking** | Log target post author/topic + whether link was included |

### Reddit / community participation

| Item | Guidance |
|------|----------|
| **What to post** | Full helpful answers; link only when sub rules allow and answer stands alone without it |
| **What not to post** | New threads that are thin ads; multiple subs same day; vote manipulation |
| **Frequency** | 2–4 quality engagements per week (not every week required on all subs) |
| **Example** | r/smallbusiness or r/freelance — answer “free PM for clients?” with framework + optional link |
| **Risk** | Medium — read sub rules; some subs ban self-promotion |
| **Tracking** | Subreddit, thread URL, removed? (Y/N) |

### Quora-style / forum answers

| Item | Guidance |
|------|----------|
| **What to post** | Direct answer to one question; structured bullets; one link at end as “further reading” |
| **What not to post** | Pasted article; duplicate answers across 20 questions |
| **Frequency** | 1–2 answers per week |
| **Example** | “Teamwork vs Asana for a 5-person agency?” — client-work vs internal framing |
| **Risk** | Low–medium |
| **Tracking** | Platform, question title, answer date |

### Community answer — task / project management

```
For a small team, start by separating “tasks” from “projects.”

Tasks = single actions with owners and due dates.
Projects = coordinated deliverables with stages.

If the pain is daily follow-through, a simple task system may be enough.
If the pain is multi-step client work, use a light project board with stages like: To Do → In Progress → Waiting on Client → Done.

A plain-language guide that walks through this:
[URL]

(Happy to adapt to your team size if you share how many people and whether work is client-facing.)
```

### Community answer — freelancer tools

```
For freelancers, the best free PM tool is usually the one you will review every Friday — not the one with the most features.

A simple setup:
• One space per client (or one board)
• Same stage names every time
• Due dates only on real deliverables

If that matches your workflow, a visual board tool often works. If you need docs + tasks together, a flexible workspace may fit better.

Overview of free options and who each fits:
[URL]
```

## 9. Decision rules

| Situation | Action |
|-----------|--------|
| **< 3 weeks since indexing / major publish** | **Wait** on fresh GSC; keep distributing useful content |
| **Page has impressions, 0 clicks, position 40–70** | **Promote more** + consider **title/meta CTR** test (one change, wait 2–3 weeks) |
| **Page has 0 impressions after 60 days** | Check indexing; light internal link support; do not spam external |
| **Community removes comment** | **Stop** that pattern; revise answer to be more standalone |
| **Referral traffic but still 0 organic clicks** | **Continue** distribution; revisit SERP snippet separately |
| **New roadmap `create_new` overlaps existing URL** | **Do not create** — deepen or distribute existing page |
| **Distribution gets replies/links** | **Promote more** on that angle (same page, new snippets) |
| **30 days distribution, zero signals** | **Stop promoting** that page temporarily; try different angle or channel |
| **Fresh GSC shows click growth** | **Update** worklog note; reduce patch urgency |
| **Fresh GSC flat, referrals growing** | **Continue** distribution; organic may lag referrals |
